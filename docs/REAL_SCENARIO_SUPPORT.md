# 问康 CareCue 泛化场景支持说明

版本：v1.0
日期：2026-06-09
状态：已完成

## 1. 背景与目标
在最初的阶段 2 引导页面中，系统仅配置了“头晕”、“胸痛”、“咳嗽”三类明确的规则原型，所有的输入如果不匹配这三者，将默认 fallback 到“头晕”场景。
为了支持真实世界中长辈复杂多变的主诉，我们对产品逻辑和前端引导进行了改造：允许用户输入任意真实问题，并且由通用场景规则作为兜底，而此前的三类典型场景保留作为辅助示例。

## 2. 修改细节

### 2.1 规则扩展与通用场景 (server/rules.ts)
*   **新增 `general` 场景**：在 `scenarios` 配置中增加了一个正则匹配兜底 `/.*/` 的 `general`（综合健康咨询）场景。
*   **新增泛化追问**：
    *   保留了通用的年龄、咨询对象提问。
    *   增加了更泛化的追问：症状出现时间（不到1天、几天了、超过1周等）、通用红旗信号（呼吸困难、剧烈胸痛、意识模糊、肢体无力、大量出血等）、其他伴随症状、既往病史和用药。
*   **修改场景识别逻辑**：`identifyScenario` 从默认 fallback `scenarios[0]` 改为了严格 fallback 到 `general` 场景。
*   **更新风险判断 `buildResult`**：在 `hasRedFlag` 检查列表中加入了新引入的泛化高危词汇 `chest_pain`（突发剧烈胸痛）和 `severe_bleeding`（大量出血、呕血或黑便）。
*   **完善综合场景输出 `buildScenarioSpecificResult`**：对于 `general` 场景，返回科室建议（全科医学科、急诊科）和方向提示（系统性疾病排查等），并提供克制的日常观察建议。

### 2.2 前端引导文本与安全提示更新 (src/App.tsx)
*   **文案更新**：将原有强调“覆盖头晕、胸痛、咳嗽三类场景”的提示，更新为“已支持真实场景。以下示例可作为引导辅助填写。”，让用户清楚系统有能力处理真实多样的输入。
*   **安全提示扩充**：右侧的“安全提醒”原本偏向最初的 Demo 场景，现已扩充为覆盖全科的高危“红旗症状”（新增了突发剧烈头痛、严重过敏等），文案更新为：“出现胸痛伴呼吸困难、突发剧烈头痛、一侧肢体无力、意识模糊、大量出血或严重过敏等情况，不建议等待线上整理结果，请优先线下急诊。”，从而在泛化场景下依然能提供严谨的医疗安全兜底。

### 2.3 测试与文档 (server/rules.test.ts, docs/PRD.md)
*   **更新测试用例**：将 fallback 测试从期望得到 `dizziness` 修改为期望得到 `general`。
*   **修改 PRD.md**：在“核心功能”及“已确认方向”的章节，增加了对“泛化的综合健康咨询场景”的支持描述，明确其兜底机制。

## 3. 验收结果
1. 原有的头晕、胸痛、咳嗽的针对性提问不受影响。
2. 任意非预设症状（如“胃有点不舒服”）均能触发通用的 `general` 提问，并在检测到高危红旗选项时，正确升级为急诊提醒。
3. 测试套件 `npm run test:rules` 及 `npm run test:api` 已全部通过。
4. 前端不引入额外依赖，通过复用既有组件完成修改。
