# 问康 CareCue 咨询入口 UI/UX 优化方案

版本：v1.0
日期：2026-06-09
状态：待开发

## 1. 背景与目标
当前咨询入口页面存在视觉压迫感较强、易引发长辈输入焦虑的问题。本方案旨在通过适老化与情感化设计，降低用户的心理门槛，使其更轻松地完成就医前症状的初步输入，符合 PRD 中“低门槛收集用户最初问题”的产品定位。

## 2. 优化细节与实现建议

### 2.1 主标题区域优化（减轻海报感与压迫感）
*   **当前问题**：标题字号过大，视觉压迫感强，长辈需要清楚而非震撼。
*   **优化方案**：缩小主标题与副标题字号，保留“先说哪里不舒服，再一步步补齐关键信息”的引导方向。
*   **开发建议**：
    *   主标题 Tailwind 类名从 `text-4xl`（或以上）降级为 `text-2xl md:text-3xl`。
    *   调整行高与字重（如 `font-medium` 或 `font-semibold`），保持阅读清晰舒适，避免大面积粗体黑体。

### 2.2 输入区域体验降压（降低心理门槛）
*   **当前问题**：大面积的多行文本框会让用户误以为需要撰写长篇大论，产生畏难情绪。
*   **优化方案**：将输入框改为中等高度，并将提示语改为更具亲和力、更口语化的场景示范。
*   **开发建议**：
    *   将 `<textarea>` 的 `rows` 属性调整为 `3` 或 `4`，设定一个适中的 `min-height`（如 `min-h-[120px]`）。
    *   更新 Placeholder 文案为：“比如：我爸最近总头晕，站起来更明显。”

### 2.3 紧急提醒卡片重构（视觉降级与动态强化）
*   **当前问题**：右侧紧急提醒面积过大、采用大块橙色/红色警示，普通场景下容易让长辈产生不必要的焦虑。
*   **优化方案**：默认状态下改为轻量级、柔和的“安全提醒卡”；仅当检测到高危场景时再动态强化视觉。
*   **开发建议**：
    *   **默认状态**：缩小卡片面积，采用浅色背景（如 `bg-gray-50` 或 `bg-slate-50`），边框柔和（`border-gray-200`），辅以温和的图标。文案以科普或普通提醒为主。
    *   **动态触发机制**：前端监听输入框内容，当匹配到“胸痛”、“呼吸困难”、“晕倒”、“吐血”、“意识异常”等红旗词汇时，卡片平滑过渡为强警示状态（如 `bg-orange-50 border-orange-200 text-orange-800`），并突出紧急就医提示。

### 2.4 快捷示例按钮（维持现状）
*   **当前判断**：示例按钮对长辈非常友好，点击即输入，无需理解复杂交互。
*   **开发建议**：严格保留现有示例按钮组件及交互逻辑，确保点击后文本能正确填入输入框，并可直接触发后续流程。

### 2.5 核心操作按钮文案（维持现状）
*   **当前判断**：“开始追问”这一文案比“发送”或“生成报告”更符合阶段 3 的产品逻辑，有助于管理用户预期，避免“AI 直接下结论”的错觉。
*   **开发建议**：维持主按钮文案为“开始追问”，并确认按钮的触控区域（确保高度 ≥ 44px），以适应长辈的操作习惯。

## 3. 验收标准
1. 页面整体视觉更加柔和，无过度夸张的字体与色块。
2. Placeholder 更新为指定口语化文案，输入框高度中等，不再占据大面积屏幕。
3. 右侧安全提醒卡在普通状态下不刺眼，输入红旗词汇后能实时切换为警示状态。
4. 示例按钮点击交互正常。
5. “开始追问”按钮触控区域符合可访问性要求。
